# artifacts/x64/ -- Mode-4 x86-64 Generated Assembly

Generated by `scrip --jit-emit --x64`. GNU-as text assembly linked against
`libscrip_rt.so`. Git history is the archive -- no session-numbered copies.
Overwrite and commit when the emitter changes.

`git log -p artifacts/x64/beauty_prog.s` shows full evolution.

## Files

    beauty_prog.s            PRIMARY: beauty.sno compiled via --jit-emit --x64
    samples/roman.s          demo/snobol4/roman.sno (recursive functions)
    samples/wordcount.s      demo/snobol4/wordcount.sno (BREAK/SPAN patterns)
    samples/claws5.s         demo/snobol4/claws5.sno (corpus-scale tokenizer)
    samples/treebank.s       demo/snobol4/treebank.sno (S-expression parser)
    smoke/                   per-rung smoke fixtures (one per opcode group)

## Update commands

Run after any change to sm_codegen_x64_emit.c, sm_macros.s, or scrip_rt.c:

    cd /home/claude/one4all
    BEAUTY=/home/claude/corpus/programs/snobol4/demo/beauty.sno

    # PRIMARY -- beauty (self-contained, 4700+ SM instructions)
    ./scrip --jit-emit --x64 $BEAUTY > artifacts/x64/beauty_prog.s
    echo "beauty: $(wc -l < artifacts/x64/beauty_prog.s) lines"

    # SAMPLES -- demo programs (tracked for visual inspection)
    ./scrip --jit-emit --x64 demo/snobol4/roman.sno    > artifacts/x64/samples/roman.s
    ./scrip --jit-emit --x64 demo/snobol4/wordcount.sno > artifacts/x64/samples/wordcount.s
    ./scrip --jit-emit --x64 demo/snobol4/claws5.sno   > artifacts/x64/samples/claws5.s
    ./scrip --jit-emit --x64 demo/snobol4/treebank.sno > artifacts/x64/samples/treebank.s

    # Verify asm is syntactically valid (assembles without error)
    for s in artifacts/x64/beauty_prog.s artifacts/x64/samples/*.s; do
        gcc -c "$s" -o /dev/null 2>/tmp/as_err.txt && echo "OK   $s" \
            || { echo "FAIL $s"; cat /tmp/as_err.txt; }
    done

    # Commit if anything changed
    git diff --stat artifacts/x64/
    git add artifacts/x64/ && git commit -m "x64 artifacts: regen after <what changed>"

## What to look for

Each emitter rung (EM-3, EM-4, ...) should visibly reduce the count of
UNHANDLED_OP markers and increase the count of real macro calls.
Scan roman.s and wordcount.s to see the emitter's current coverage.

Milestone check for EM-7 (beauty.sno oracle gate):
    - beauty_prog.s must contain zero UNHANDLED_OP lines
    - assembled binary output must match SPITBOL md5 abfd19a7a834484a96e824851caee159
